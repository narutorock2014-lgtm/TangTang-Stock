# 🍜 TangTang-Stock — Stock Management

เว็บแอปจัดการสต็อกวัตถุดิบในครัว บันทึกยอดยกมา/ลงของเพิ่ม/เดลิเวอรี่/ขึ้นบาร์รายวัน
คำนวณคงเหลือ สถานะสั่งซื้อ export เป็น Excel และดูกราฟรายงานได้

ไฟล์เดียว (`index.html`) ไม่ต้องมีเซิร์ฟเวอร์ ใช้งานได้ทันทีแค่เปิดในเบราว์เซอร์
หรือจะอัปโหลดขึ้น **GitHub Pages** ให้เข้าจากมือถือ/คอมทุกเครื่องก็ได้ (ดูขั้นตอนด้านล่าง)

---

## 1) วิธีเอาขึ้น GitHub Pages (ฟรี)

1. สร้าง repository ใหม่บน GitHub เช่นชื่อ `kitchen-stock`
2. อัปโหลดไฟล์ `index.html` (และ `README.md`) เข้า repo — ลากไฟล์วางในหน้าเว็บ GitHub
   ได้เลย หรือใช้คำสั่ง:
   ```
   git init
   git add index.html README.md
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/<ชื่อผู้ใช้>/kitchen-stock.git
   git push -u origin main
   ```
3. เข้า **Settings → Pages** ของ repo
4. ที่ "Build and deployment" เลือก Source = `Deploy from a branch`,
   Branch = `main`, folder = `/(root)` แล้วกด Save
5. รอ 1–2 นาที จะได้ลิงก์ประมาณ
   `https://<ชื่อผู้ใช้>.github.io/kitchen-stock/`
6. เปิดลิงก์นี้จากมือถือหรือคอมเครื่องไหนก็ได้ — จะเห็นหน้าแอปเหมือนกัน
   (บันทึกคำสั่งเป็น Add to Home Screen บนมือถือได้ ใช้เหมือนแอป)

> หมายเหตุ: ถ้าไม่เปิดใช้ Firebase (ข้อ 2) ข้อมูลแต่ละเครื่อง/เบราว์เซอร์จะ
>**แยกกัน** เก็บในเครื่องนั้นเท่านั้น (IndexedDB) ไม่เห็นข้อมูลของกันและกัน

## 2) วิธีเปิดใช้ "ซิงก์ข้อมูลเรียลไทม์ข้ามเครื่อง" (Firebase — ฟรี)

ต้องการให้เครื่อง A แก้ไข แล้วเครื่อง B เห็นข้อมูลล่าสุดทันที ต้องมีฐานข้อมูลกลาง
บนคลาวด์ ในไฟล์นี้เตรียมโค้ดเชื่อมต่อ **Firebase Firestore** ไว้ให้แล้ว (แผนฟรี
"Spark" เพียงพอสำหรับใช้งานร้านเล็ก-กลาง) แค่กรอกค่า config ของคุณเอง:

1. ไปที่ https://console.firebase.google.com → สร้างโปรเจกต์ใหม่ (ฟรี)
2. ในโปรเจกต์ กด **Build → Firestore Database → Create database**
   เลือกโหมด "Start in test mode" ก่อน (ปรับ Security Rules ทีหลังได้ ดูข้อ 4)
3. กด ⚙️ (Project settings) → เลื่อนลงหา "Your apps" → กด ไอคอนเว็บ `</>`
   ตั้งชื่อแอปอะไรก็ได้ แล้วจะได้ config object หน้าตาแบบนี้:
   ```js
   const firebaseConfig = {
     apiKey: "AIza...",
     authDomain: "xxxx.firebaseapp.com",
     projectId: "xxxx",
     storageBucket: "xxxx.appspot.com",
     messagingSenderId: "...",
     appId: "..."
   };
   ```
4. เปิดไฟล์ `index.html` หาคำว่า `FIREBASE_CONFIG` (อยู่ช่วงต้นของ `<script>`)
   แล้ววางค่าที่ได้จากข้อ 3 แทนที่ `YOUR_API_KEY` ฯลฯ ให้ครบทุกช่อง
5. เซฟไฟล์ แล้ว push ขึ้น GitHub ใหม่อีกครั้ง (หรือถ้ายังไม่ได้ deploy ก็ push
   ตามข้อ 1)
6. เปิดเว็บอีกครั้ง ที่แถบใต้ช่องค้นหาจะขึ้นจุดสีเขียว "ซิงก์แบบเรียลไทม์ —
   เชื่อมต่อแล้ว" แปลว่าใช้งานได้ ทุกเครื่องที่เปิดลิงก์เดียวกันจะเห็นข้อมูล
   เดียวกันและอัปเดตให้อัตโนมัติเมื่อมีการแก้ไขจากเครื่องไหนก็ตาม

### 3) Firestore Rules (ไม่มีระบบล็อกอินแล้ว — ทุกคนที่มีลิงก์แก้ข้อมูลได้)

เวอร์ชันนี้**ตัดระบบล็อกอินออกแล้ว** เพื่อความง่ายในการใช้งาน ให้ตั้ง Firestore
Rules แบบเปิดไว้ (ไม่ต้องล็อกอิน อ่าน/เขียนได้เลย):

1. Firebase Console → Firestore Database → แท็บ **Rules**
2. แทนที่กฎเดิมทั้งหมดด้วยข้อความนี้:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /shops/{shopId}/{document=**} {
         allow read, write: if true;
       }
     }
   }
   ```
3. กด **Publish**

⚠️ **ข้อควรระวัง:** กฎนี้เปิดให้**ใครก็ตามที่รู้ลิงก์เว็บ**สามารถดู/แก้ไขข้อมูลสต็อกได้
โดยไม่ต้องล็อกอิน เหมาะสำหรับใช้งานภายในร้านที่ไม่ได้กังวลเรื่องคนนอกเข้าถึงลิงก์
ถ้าภายหลังอยากเพิ่มความปลอดภัยอีกครั้ง (เช่นระบบรหัสผ่านร้าน) แจ้งได้เลย

ถ้าเคยตั้ง Firebase Authentication (Email/Password) ไว้จากเวอร์ชันก่อนหน้า จะยังอยู่ในระบบ
เฉยๆ ไม่ได้ใช้งานแล้ว ไม่ลบก็ได้ไม่มีผลอะไร

### ถ้ายังไม่กรอก Firebase config
แอปจะทำงานในโหมดออฟไลน์ (เดิม) ทุกอย่างเก็บใน IndexedDB ของเบราว์เซอร์นั้น ๆ
ใช้งานได้ปกติทุกฟีเจอร์ เพียงแต่ไม่ซิงก์ข้ามเครื่อง — ไม่ต้องกลัวพัง

## ฟีเจอร์หลัก

- บันทึกยอดยกมา / ลงของเพิ่ม / เดลิเวอรี่ / ขึ้นบาร์ รายวัน แยกตามหมวดหมู่สินค้า
- คอลัมน์ **"รวมใช้ไปวันนี้"** (เดลิเวอรี่ + ขึ้นบาร์) คำนวณสดทุกครั้งที่กรอก
- คำนวณคงเหลือ, จำนวนวันที่ใช้ได้อีก, สถานะ (ปกติ/ใกล้สั่ง/สั่งซื้อ/หมดสต็อก) อัตโนมัติ
- ล็อกข้อมูลของวันที่ผ่านมาไม่ให้แก้ไขพลาด (กดปลดล็อกได้ถ้าจำเป็น)
- Export Excel รายวัน/สัปดาห์/เดือน — ทุกแถวมีคอลัมน์ **"วันที่"** กำกับชัดเจน
  แยกชีตตามหมวดหมู่ + มีชีตสรุปรวมทุกหมวด
- หน้ารายงาน/กราฟสรุปการใช้วัตถุดิบ (Chart.js) ย้อนหลัง 1/7/30 วัน
- สำรอง/กู้คืนข้อมูลทั้งหมดเป็นไฟล์ JSON
- โหมดมืด/สว่าง
- ซิงก์ข้อมูลเรียลไทม์ข้ามอุปกรณ์ผ่าน Firebase (ถ้าตั้งค่าตามข้อ 2)
